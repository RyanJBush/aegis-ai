# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| latest (`main`) | ✅ |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report them via email to **ryanjbush@gmail.com** with the subject line `[SECURITY] Obsidian — <brief description>`.

Include:
- A description of the vulnerability and its potential impact
- Steps to reproduce
- Any suggested remediation if known

You can expect an acknowledgement within **48 hours** and a resolution timeline within **7 days** for critical issues.

## Scope

This project is a portfolio/demonstration platform. It intentionally includes **intentionally vulnerable demonstration targets** for scanner testing. These are scoped, sandboxed components — not production services. All real application code follows secure coding standards.

## Responsible Disclosure

If you discover a vulnerability in the **scanner or platform code itself** (not the demo targets), please disclose responsibly using the contact above.
